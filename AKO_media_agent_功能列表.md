# AKO Media Agent 项目功能列表

> 版本：v1.2 | 整理日期：2026-07-22 | 基于代码、配置文件及文档分析

---

## 一、五层核心架构总览

| 层级 | 名称 | 职责 |
|------|------|------|
| Layer 1 | 数据收集层 | 7大平台数据采集、RSS新闻监控、竞品追踪、趋势探测、采集合规 |
| Layer 2 | 智能分析层 | 情感分析、主题聚类、关键词提取、互动预测、竞品缺口分析 |
| Layer 3 | 决策引擎 | 内容规划、标题优化、发布时间优化 |
| Layer 4 | 执行层 | AI文案生成、视觉配图、多平台发布、小程序内容同步/推荐/SEO |
| Layer 5 | 反馈闭环 | 表现追踪、策略自动调整、异常检测与回退 |

## 二、Layer 1：数据收集层

### 2.1 社交媒体全量采集（MediaCrawlerWrapper）

- 支持 **7大平台** 数据抓取：小红书、抖音、微博、B站、知乎、公众号、视频号
- 基于 Playwright 浏览器自动化 + MediaCrawler 开源框架（MIT协议）
- 异步爬虫架构，支持代理 IP 轮换（Bright Data / DataImpulse）
- 多账号池轮换采集（每平台5个账号）
- 自动数据清洗与格式化
- 数据存储至数据库（MySQL/SQLite）
- 每日采集统计报表生成

### 2.2 RSS 新闻监控（NewsMonitor）

- 自动监控预设 RSS 源（中国建筑新闻网、住建部官网）
- 关键词匹配识别相关新闻（装配式建筑、陶粒混凝土、预制构件、T/CECS 10154、模块化建筑、钢结构住宅、绿色建材、碳中和建筑、装配式政策、建筑工业化、智能建造）
- LLM 自动生成新闻摘要
- 行业政策变动自动检测与告警（邮件/SMS 通知）
- 24小时内新闻回溯查询

### 2.3 竞品追踪（CompetitorTracker）

- 支持 4 家竞品追踪：远大住工、中民筑友、美好置业、三一筑工
- 竞品内容自动记录（平台、主题、互动数据）
- 爆款内容识别（互动 > 均值3倍）
- 竞品动态摘要生成
- 定时提醒检查（每周五）

### 2.4 趋势探测（TrendDetector）

- **4大趋势数据源**：百度指数、微信指数、抖音热榜、小红书热搜
- **三级关键词矩阵**：
  - Tier 1：装配式建筑、模块化住宅
  - Tier 2：陶粒混凝土、预制墙板、快速建房
  - Tier 3：农村自建房、轻钢别墅、集装箱房
  - 长尾：贵阳装配式建筑、贵州模块化建筑、三层别墅造价
- 热度指数获取与排名记录
- 关键词趋势曲线查询（30天）

### 2.5 采集合规模块（Compliance）

- **ProxyManager**：代理 IP 管理（Bright Data / DataImpulse）
- **RateLimiter**：请求频率控制（最小间隔2秒，随机抖动±0.5秒，日请求上限500条/平台）
- **AccountPool**：多账号轮换
- **CaptchaHandler**：验证码处理
- 活跃时段限制（09:00-23:00）
- 仅采集公开内容，自动脱敏用户隐私数据

## 三、Layer 2：智能分析层

### 3.1 情感分析（SentimentAnalyzer）

- **三级引擎**：jieba 本地规则引擎（免费）→ 阿里云 NLP API（高精度）→ DeepSeek LLM（最准确）
- 批量文本情感分析
- 品牌情感全景分析
- 舆情危机预警（负面比例 > 30% 或负面数量 > 10）
- 自动告警：邮件+SMS，响应时间承诺30分钟

### 3.2 主题聚类（TopicCluster）

- 双重算法：
  - **BERTopic**（需安装 bertopic，效果最佳）
  - **TF-IDF + KMeans**（默认回退方案，零额外依赖）
- 中文分词（jieba）
- 10~50 个主题自动聚类
- 主题分布统计
- 内容缺口发现

### 3.3 关键词提取（KeywordExtractor）

- 双重算法：**TF-IDF** + **TextRank**
- 结果合并排序（Top 50）
- 基于近期内容自动提取
- 词云数据生成
- SEO 关键词建议

### 3.4 互动预测（EngagementPredictor）

- 三阶段演进路线：
  - Phase 1：规则基线公式（`followers × base_rate × title_score × hour_coefficient`）
  - Phase 2：轻量 ML 模型
  - Phase 3：时序预测模型
- 标题质量评分（引爆力、情绪词、关键词密度等维度）
- 分平台基础互动率预设

### 3.5 竞品缺口分析（GapAnalyzer）

- 竞品内容覆盖度矩阵
- AKO 内容覆盖度矩阵
- 缺口识别与机会排序
- 热度图数据输出

## 四、Layer 3：决策引擎

### 4.1 内容规划器（ContentPlanner）

- **5条规划规则**：
  1. **热点响应**：热度 > 阈值，2小时内响应
  2. **竞品跟进**：竞品爆款触发，48小时内响应
  3. **常青排期**：按固定排期产出
  4. **品牌事件**：项目节点/里程碑触发
  5. **政策响应**：行业政策变动，4小时内响应
- 日计划 / 周计划 / 月计划生成
- 规划结果存储至数据库

### 4.2 标题优化器（TitleOptimizer）

- **6大标题公式**：时间切片、数字诚实、材料对话、人物在场、技术真相、失败档案
- 每条选题生成 3 个候选标题
- 三引擎评分：点击率引擎 + 搜索引擎 + 社交引擎
- 候选标题对比排序

### 4.3 发布时间优化器（TimingOptimizer）

- 分平台最佳时段推荐：
  - 公众号：周二/四 18:00-20:00
  - 小红书：周三/五 12:00-14:00，每天 19:00-21:00
  - 抖音：每天 18:00-22:00，周末全天
  - LinkedIn：周二/三 08:00-10:00
  - 博客：周二/四 10:00-12:00
- 历史表现数据分析
- 最优时段自动排列
- 完整定时报告

## 五、Layer 4：执行层

### 5.1 AI 文案生成（Copywriter）

- 对接 DeepSeek `deepseek-chat` 模型
- 品牌风格体系注入：**切片-展开-收束** 结构
- 品牌语料库风格控制（最少100篇样本，一致性阈值80%）
- 分平台写作参数适配：
  - 公众号：800-2000字，深度+故事风格
  - 小红书：300-600字，口语+种草风格
  - 抖音：15-60秒脚本，短句+节奏
  - LinkedIn：200-500字英文，专业+数据
  - 博客：1500-3000字，SEO+深度
- 风格一致性自动检测
- 思想厚度自动评分（不可替代性40% + 具体性30% + 时间诚实15% + 人的在场15%）
- 单篇生成 + 多平台一键适配

### 5.2 视觉生成（VisualGenerator）

- 对接 AKO-ART-Agent 视觉生成接口
- 品牌色系锁定：奶油金 #EBDAB9、深棕黑 #231E1C、琥珀金 #A08C64、熔金 #B99B5F
- 最低分辨率 1920x1080
- 色彩合规自动检测
- 分平台尺寸适配（公众号 900x383，小红书 3:4 比例等）

### 5.3 多平台发布（Publisher）

- **自动发布**：公众号（通过微信桥接 API）、博客（akobuild.cloud）
- **半自动发布**（生成内容需人工确认）：小红书、抖音、LinkedIn
- 内容草稿版本管理
- 发布记录持久化
- 内容撤回功能
- 分阶段审核模式支持（Phase 1 100%人工 → Phase 2 50%抽检 → Phase 3 85%+自动）

### 5.4 小程序模块（MiniProgram）

- **ContentSync**：公众号内容 → 小程序自动同步（文章、案例、产品）
- **SEOOptimizer**：标题优化、关键词提取、描述生成、SEO评分
- **Recommendation**：用户个性化内容推荐、推荐点击追踪
- **Monitor**：小程序数据看板、异常告警
- **FormTrigger**：表单提交智能解析、用户需求匹配、案例推荐

## 六、Layer 5：反馈闭环

### 6.1 表现追踪（PerformanceTracker）

- 追踪指标：阅读量、点赞、评论、转发、收藏、完读率、粉丝增长、转化率、小程序数据
- 单篇表现数据记录
- **周报自动生成**（每周一 09:00）
- **月报自动生成**（每月1日 09:00）
- Top-N 优质内容排行

### 6.2 策略自动调整（StrategyAdjuster）

- 内容类型表现分析
- 自动建议调整维度：内容类型权重、发布时间、关键词优先级
- 策略调整历史记录（30天回溯）
- 自动化率随阶段逐步提升

### 6.3 异常回退（RollbackManager）

- 自动检测异常类型：
  - 负面评论比例 > 50%
  - 阅读量 < 预估的 20%
  - 平台下架触发
  - 用户投诉触发
- 异常告警通知（邮件）
- 内容自动撤回
- 撤回原因分析

## 七、进化模块（Evolution）

### 7.1 记忆层（Memory Layer）

- **ContentDNA**：内容基因提取（结构基因 + 风格基因 + 思想基因），基因分类（优势/中性/弱势），主导基因排行
- **AKOCognitionGraph**：品牌认知图谱（实体提取、认知纠错、认知更新报告）
- **FailureCaseLibrary**：失败案例库（记录、风险检测、规避规则生成、失败模式分析）
- **ReaderEvolution**：读者进化追踪（兴趣迁移检测、偏好更新、需求预测）

### 7.2 学习层（Learning Layer）

- **CompetitorPatternLearning**：竞品模式学习
- **PerformanceDeepLearning**：表现深度学习（内容DNA × 表现数据 → 规律挖掘）
- **FeedbackSemanticLearning**：反馈语义学习（评论内容 → 深层洞察）
- **IndustryKnowledgeGraph**：行业知识图谱（核心节点初始化、自动扩展、图谱摘要）

### 7.3 涌现层（Emergence Layer）

- **AutoToolDevelopment**：自动工具开发（工作流瓶颈识别 → 需求 → 新工具建议）
- **ContentFormDiscovery**：内容形式发现（高表现内容共性提取、新内容形式建议）
- **ReaderCoCreation**：读者共创（问题→内容、想法→内容、评论→内容）
- **CrossDomainTransfer**：跨领域迁移（建筑 × 哲学/艺术/科技/社会等视角融合）

## 九、审核与合规体系

### 9.1 五级审核流程

| 级别 | 名称 | 类型 | 标准 |
|------|------|------|------|
| Level 1 | AI自审 | 自动 | 风格一致性(80分)、数据准确性 |
| Level 2 | AI思想厚度初评 | 自动 | 不可替代性≥60分 |
| Level 3 | 运营初审 | 人工 | 品牌调性、合规性 |
| Level 4 | 思想厚度终审 | 人工（品牌主理人） | ≥75分 |
| Level 5 | 发布前静默期 | 自动 | 24小时冷却 |

### 9.2 红线规则

- 思想厚度终审必须由人完成
- 灵魂元素必须由人提供
- 发布前必须有24小时静默期
- 互动预测仅作内部参考
- 不采集付费/会员内容

### 9.3 合规文档

- 法务审查清单（`compliance/legal_review.md`）
- 数据使用政策（`compliance/data_usage_policy.md`）
- 应急响应预案（`compliance/incident_response.md`）
