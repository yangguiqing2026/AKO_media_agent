## 1. 概述

### 1.1 定位

AKO_media_agent 是 AKO 品牌体系的**自主媒体数据智能体**，核心能力覆盖：
- **自主数据采集**：7×24小时无人值守抓取社交媒体、行业新闻、竞品动态、趋势数据
- **智能分析**：NLP情感分析、主题聚类、关键词提取、互动预测
- **数据驱动决策**：自动生成内容策略、选题、标题、发布时间
- **自动执行**：生成文案、配图、发布到各平台
- **反馈优化**：追踪表现数据，自动调整策略参数

**核心原则**：**思想厚度 > 传播效率，品牌高度 > 流量规模**

AKO_media_agent 不是"内容生产工具"，是**思想表达的放大器**。它的核心任务不是让AKO发更多内容，是让AKO的**每一句话都更不可替代**。

**技术服务于思想，而非思想服务于技术。**

### 1.2 设计哲学

> "感知 → 理解 → 决策 → 执行 → 反馈" 闭环

**分阶段实现路径**：
- **Phase 1（前3月）**：AI辅助模式——AI生成成品，100%人工审核
- **Phase 2（3-6月）**：AI主导模式——AI生成+抽检审核，自动化率60%
- **Phase 3（6-12月）**：AI自主模式——AI生成+异常触发审核，自动化率85%+

### 1.3 内容哲学：只有AKO能说的话

每一条内容必须通过测试：

> **"如果远大住工、中民筑友也发这条内容，会不会显得合理？"**
> 
> 如果答案是"会"，那这条内容就不应该发。

AKO的内容厚度来自三个不可替代的源头：
1. **建造现场的真实** —— 不是看工地，是站在吊车上的人
2. **技术参数的诚实** —— 138kg/m²到底意味着什么
3. **个人生命经验的投射** —— 心梗后的思维转变、对建筑的理解、对孤独的坦然

**情绪公式**：
> 精致静谧 = 暖金统治 + 白色软化 + 黑色沉稳 + **人的在场** + **时间的诚实**

### 1.4 与现有 Agent 的关系

```
AKO_layout_agent  →  效果图/方案  →  AKO_media_agent  →  项目推文
AKO_quote_agent   →  报价数据    →  AKO_media_agent  →  "造价说"系列
AKO_hub           →  项目进度    →  AKO_media_agent  →  "工地日记"

微信小程序        ←  内容同步    ←  AKO_media_agent  →  用户行为回流
```

### 3.3 合规运行措施

**MediaCrawler 开源协议（MIT）允许商业使用，但需遵守平台规则**：

1. **住宅代理IP池**
   - 提供商：Bright Data / DataImpulse
   - 轮换间隔：≥2秒/请求
   - 地理分布：分散在多个省市
   - 自动切换：单IP连续失败3次自动切换

2. **多账号轮换机制**
   - 账号池：5个账号（不同手机号注册）
   - 轮换策略：轮询使用，单账号日请求≤100次
   - 自动检测：登录态失效时自动切换+邮件告警
   - 备用账号：主账号封禁时自动启用备用

3. **请求频率控制**
   - 最小间隔：2秒/请求
   - 随机抖动：±0.5秒
   - 时段分布：模拟真人活跃时段（09:00-23:00）
   - 日总量上限：单平台≤500条

4. **数据采集边界**
   - ✅ 仅采集公开页面（不登录抓取私密内容）
   - ✅ 不绕过反爬机制（遇到验证码自动暂停）
   - ✅ 不采集用户隐私信息（手机号、地址等）
   - ✅ 仅用于学习研究和品牌监测，不售卖数据
   - ❌ 不采集付费/会员内容
   - ❌ 不批量采集用户关系链

5. **法务审查**
   - 上线前法务出具书面合规意见
   - 保留所有采集日志（6个月）
   - 建立投诉响应机制（24小时内处理）

### 4.4 互动预测模型（分阶段实现）

**分阶段路径**：

| 阶段 | 时间 | 方法 | 准确率 | 用途 |
|------|------|------|--------|------|
| Phase 1 | 前3月 | 规则基线 | ~60% | 内部参考 |
| Phase 2 | 3-6月 | 轻量ML（线性回归/随机森林） | ~70% | 内部参考 |
| Phase 3 | 6-12月 | 时序模型（ARIMA/LSTM） | ~75% | 内部参考 |

**规则基线公式（Phase 1）**:
```python
def predict_engagement(followers, title_score, publish_hour, platform):
    base_rate = {
        "wechat": 0.05,      # 公众号打开率5%
        "xiaohongshu": 0.10,  # 小红书曝光率10%
        "douyin": 0.15,      # 抖音播放率15%
        "linkedin": 0.08,    # LinkedIn打开率8%
        "blog": 0.03         # 博客访问率3%
    }

    predicted_reads = followers * base_rate[platform] * title_score * hour_coefficient[publish_hour]
    predicted_likes = predicted_reads * 0.08  # 点赞率约8%
    predicted_comments = predicted_reads * 0.015  # 评论率约1.5%

    return predicted_reads, predicted_likes, predicted_comments
```

**重要声明**：预测结果**仅内部参考**，不对外宣称，不用于自动决策

## 5. Layer 3: 决策引擎

### 5.1 内容规划器 (Content Planner)

**输入**: 趋势数据 + 竞品分析 + 品牌日历 + 项目节点
**输出**: 7日/30日内容排期表

**规则**:

| 规则 | 触发条件 | 响应时间 | 输出 |
|------|----------|----------|------|
| 热点响应 | 检测到高热度话题（热度>阈值） | 2小时内 | 选题建议 + 初稿 |
| 竞品跟进 | 竞品爆款内容（互动>均值3倍） | 48小时内 | 差异化版本 |
| 常青内容 | 技术深度文（固定节奏） | 按排期 | 排期提醒 |
| 品牌事件 | 项目节点/里程碑 | 自动触发 | 内容生成 |
| 政策响应 | 行业政策变动 | 4小时内 | 解读文章 |

### 5.2 标题优化器 (Title Optimizer)

**方法**: A/B测试历史数据 + 情感分析

**输出**:
- 3个备选标题
- 预估点击率排名
- 优化建议

**标题公式库（思想维度版）**:
```
时间切片: 时间 + 地点 + 具体动作
        "6:17，贵阳，吊车还没醒，但老周已经在了"

数字诚实: 数字 + 身体感知 + 时间
        "138kg/m²，三个人，黄昏"

材料对话: 材料 + 情绪 + 问题
        "混凝土在下午五点半为什么变温柔？"

人物在场: 人名 + 动作 + 一句话
        "老周：下个月能收工，但不知道风还大不大"

技术真相: 标准号 + 行业不愿说的事
        "T/CECS 10154：为什么很多墙板通不过"

失败档案: 时间 + 失败 + 学到的事
        "2024年3月，那批墙板返工了"
```

## 6. Layer 4: 执行层

### 6.1 自动文案生成（思想维度版）

**输入**: 选题 + 数据洞察 + 品牌模板
**输出**: 多平台适配文案

**分阶段审核策略**：

| 阶段 | 时间 | 审核方式 | 自动化率 | 目标 |
|------|------|----------|----------|------|
| Phase 1 | 前3月 | 100%人工审核AI成品 | 0% | 建立品牌语料库，训练AI风格 |
| Phase 2 | 3-6月 | 抽检审核（50%） | 50% | 验证AI质量，逐步提效 |
| Phase 3 | 6-12月 | 异常触发审核 | 85%+ | AI主导，人工兜底 |

**风格控制（思想维度版）**:
- 建立**AKO品牌语料库**（100篇以上优质人工文案作为训练样本）
- 使用**few-shot prompt + 风格约束模板**
- 设置**风格一致性评分**（人工标注100篇，AI生成后自动对比，低于80分重写）

**核心句式**：不是"我们提供…"，是"这个下午…"

| 常规写法 | AKO写法 |
|----------|---------|
| "AKO提供装配式建筑解决方案" | "这个下午，第17个模块在桐木岭就位" |
| "我们的墙板抗压强度达到5-10MPa" | "5-10MPa，意思是每平方厘米能承受50-100公斤——三个人站在上面，没晃" |
| "我们致力于推动建筑工业化" | "老周抽了根烟，说下个月能收工" |

**段落结构**：切片-展开-收束
```
1. 切片（1句）：一个具体的瞬间或数字
   "第17个模块，重4.2吨。"

2. 展开（3-5句）：技术/人文/时间维度
   "吊车臂展35米，操作员老周说今天风大，放慢了速度。
    墙板厚度120mm，面密度138kg/m²，螺栓连接。
    三个人站在顶板上验收，没晃。"

3. 收束（1句）：回到人，或回到时间，或回到无法回答的问题
   "老周抽了根烟，说下个月能收工。"
   或
   "但谁知道下个月，风还会不会这么大？"
```

**平台适配**:

| 平台 | 字数 | 风格 | 特殊要求 |
|------|------|------|----------|
| 公众号 | 800-2000字 | 深度+故事 | 封面图900×383px |
| 小红书 | 300-600字 | 口语+种草 | 首图3:4，关键词# |
| 抖音 | 15-60秒脚本 | 短句+节奏 | 字幕+BGM建议 |
| LinkedIn | 200-500词 | 专业+数据 | 英文，项目案例 |
| 官网博客 | 1500-3000字 | SEO+深度 | 关键词布局、内链 |

## 9. 审核流程（思想维度版）

### 9.1 五级审核体系

```
AI生成
  ↓
【Level 1】AI自审：风格一致性 + 数据准确性
  ↓（通过≥80分）
【Level 2】AI思想厚度初评：不可替代性测试
  ↓（通过≥60分）
【Level 3】运营初审：品牌调性 + 合规性
  ↓（通过）
【Level 4】思想厚度终审：品牌负责人（用户本人）
  ↓（通过≥75分）
【Level 5】发布前静默期：24小时冷却
  ↓
发布
```

### 9.2 思想厚度评分（0-100分）

| 维度 | 权重 | 说明 |
|------|------|------|
| **不可替代性** | 40% | 内容中"只有AKO能说的元素"占比 |
| **具体性** | 30% | 具体数字/具体人/具体场景出现频率 |
| **时间诚实** | 15% | 是否经得起时间检验 |
| **人的在场** | 15% | 是否有人的在场感 |

**发布标准**：思想厚度评分 ≥ 75分

### 9.3 核心测试（终审，必须由人完成）

1. **"远大住工会发这个吗？"** → 如果会，不发
2. **"没有AKO的具体现场/数字/人，能成立吗？"** → 如果能，不发
3. **"3年后看，还成立吗？"** → 如果不成立，慎重发
4. **"这条内容让我自己感到陌生吗？"** → 如果陌生，不发
5. **"这条内容说出了我想到但没说清楚的事吗？"** → 如果是，这是好内容

### 9.4 三条不可逾越的红线

1. **思想厚度终审必须由人完成** —— "只有AKO能说的话"只有AKO的人能判断
2. **灵魂元素必须由人提供** —— AI可以放大，但不能创造
3. **发布前必须有24小时静默期** —— 防止冲动，确保内容经得起时间检验

## 12. 安全与合规

### 12.1 数据安全

- 社交媒体数据仅采集**公开信息**，不抓取登录后私密内容
- 用户隐私数据脱敏处理（手机号、地址等字段删除）
- 敏感凭证存储于 `secrets.env`，通过 `os.getenv` 加载
- 符合《数据安全法》《个人信息保护法》
- 采集日志保留6个月，备查

### 12.2 爬虫合规（MediaCrawler MIT开源）

**法务审查清单**：
- [ ] 上线前法务出具书面合规意见
- [ ] 明确数据采集边界（公开信息、学习研究、不售卖）
- [ ] 建立投诉响应机制（24小时内处理）
- [ ] 保留所有采集日志（6个月）
- [ ] 定期审查爬虫行为（每月）

**技术合规措施**：
- MediaCrawler 使用 Playwright 模拟真实浏览器，**非逆向工程**
- 遵守各平台 robots.txt
- 控制请求频率（≥2秒/次，随机抖动±0.5秒）
- 使用代理IP分散请求
- 多账号轮换，单账号日请求≤100次
- 仅用于学习研究和品牌监测，不用于商业数据售卖
- 遇到验证码自动暂停，人工介入

### 12.3 品牌安全

- 小红书/抖音/LinkedIn内容发布前**人工审核**
- 负面舆情相关的内容需**人工审核**
- 危机预警：负面情感超过30%阈值时**自动邮件告警**
- 竞品内容不直接引用，仅做分析参考

### 12.4 异常回退

**触发条件**：
- 发布后1小时内负面评论>50%
- 发布后阅读量<预估值20%
- 内容被平台下架/限流
- 收到用户投诉

**回退流程**：
```
1. 自动检测异常
2. 邮件告警运营人员
3. 人工确认
4. 一键撤回（基于版本化存储）
5. 分析原因
6. 反馈至策略调整器
```

## 16. 最终结论

AKO_media_agent 的战略愿景正确，五层架构合理，与AKO自有Agent生态的耦合是最大差异化优势。

**核心原则**：
1. **思想厚度 > 传播效率**：每一句话都更不可替代
2. **品牌高度 > 流量规模**：让AKO的内容成为行业参照
3. **技术服务于思想**：AI放大思考，不替代思考
4. **加固而非降级**：在保持自主愿景的前提下，补齐合规、接口、异常处理、分阶段实现四个短板
5. **分阶段实现**：15周MVP，6个月达到"AI主导"，12个月接近"AI自主"

**关键红线**：
- ❌ 思想厚度终审必须由人完成
- ❌ 灵魂元素必须由人提供（风的感觉、烟的味道、眼神的意思）
- ❌ 发布前必须有24小时静默期
- ❌ 互动预测仅作内部参考，不得用于对外宣称
- ❌ 不采集付费/会员内容，不售卖数据

**MediaCrawler 开源免费降低了成本，但没有降低标准。** 预算从11.1万/年降至~15万/年（不含可选商业API），但合规措施、思想厚度审核、分阶段实现策略不减。

AKO有实力打造行业领先的AI媒体中台，但需尊重技术客观规律、法律边界，以及**思想不可替代的本质**。
